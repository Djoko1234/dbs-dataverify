## 🔐 SÉCURITÉ — COMPRENDRE L'ARCHITECTURE

### Comment fonctionnent les mots de passe

```
Mot de passe tapé par l'utilisateur
         ↓
    hashPassword() — SHA-256 (crypto.subtle — natif navigateur)
         ↓
Hash SHA-256 comparé avec USERS_DB[i].password
         ↓
Match + active===true → Connexion autorisée
```

> ⚠️ **IMPORTANT** : Changer ces mots de passe dès le premier déploiement via la section Admin → Utilisateurs → Modifier.

### Comment la désactivation fonctionne maintenant

```
Admin clique "Désactiver"
         ↓
toggleUser() modifie APP.users[x].active = false
         +
USERS_DB[x].active = false  ← SOURCE DE VÉRITÉ LOGIN
         ↓
Vérification toutes les 30 secondes pour toute session active
         ↓
Si compte désactivé détecté → doLogout() forcé immédiatement
         ↓
Au prochain login → doLogin() vérifie USERS_DB → Accès refusé
```

### Installation rapide

### Fichier backend/main.py (essentiel)

```python
from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
import oracledb, pandas as pd, json, os
from datetime import datetime

# Mode thick pour Oracle ANO encryption
oracledb.init_oracle_client(lib_dir=r"C:\oracle\instantclient_21_10")

app = FastAPI(title="DBS DataVerify API")
app.add_middleware(CORSMiddleware, allow_origins=["https://djoko1234.github.io"], allow_methods=["*"], allow_headers=["*"])

class ConnParams(BaseModel):
    host: str; port: int = 1521; service: str
    username: str; password: str; db_type: str = "oracle"

class ExecRequest(BaseModel):
    prod: ConnParams; rest: ConnParams | None = None
    at_date: str; queries: list; mode: str = "both"

@app.post("/api/execute")
async def execute(req: ExecRequest):
    results = []
    conn_prod = oracledb.connect(user=req.prod.username, password=req.prod.password,
                                  dsn=f"{req.prod.host}:{req.prod.port}/{req.prod.service}")
    for q in req.queries:
        sql = q['sql'].replace(':at_date', f"TO_DATE('{req.at_date}','YYYY-MM-DD')")
        try:
            cursor = conn_prod.cursor()
            cursor.execute(sql)
            cols = [d[0].lower() for d in cursor.description]
            prod_rows = [dict(zip(cols, row)) for row in cursor.fetchall()]
            rest_rows = []
            if req.mode == 'both' and req.rest:
                conn_rest = oracledb.connect(user=req.rest.username, password=req.rest.password,
                                             dsn=f"{req.rest.host}:{req.rest.port}/{req.rest.service}")
                c2 = conn_rest.cursor(); c2.execute(sql)
                rest_rows = [dict(zip([d[0].lower() for d in c2.description], row)) for row in c2.fetchall()]
                conn_rest.close()
            results.append({"query": q['name'], "prod": prod_rows, "rest": rest_rows,
                           "status": "OK" if prod_rows == rest_rows else "ECART"})
        except Exception as e:
            results.append({"query": q['name'], "error": classify_error(str(e)), "status": "ERROR"})
    conn_prod.close()
    return {"results": results, "at_date": req.at_date, "run_at": datetime.now().isoformat()}

def classify_error(err: str) -> dict:
    """Classifier les erreurs Oracle pour diagnostic clair"""
    if "ORA-12541" in err: return {"code": "ORA-12541", "msg": "Serveur inaccessible — Vérifier VPN et réseau", "type": "RÉSEAU"}
    if "ORA-01017" in err: return {"code": "ORA-01017", "msg": "Identifiant ou mot de passe invalide", "type": "AUTH"}
    if "ORA-12154" in err: return {"code": "ORA-12154", "msg": "Service/SID introuvable — Vérifier le nom de service", "type": "CONFIG"}
    if "ORA-00942" in err: return {"code": "ORA-00942", "msg": "Table ou vue inexistante — Droits insuffisants", "type": "DROITS"}
    if "timeout" in err.lower(): return {"code": "TIMEOUT", "msg": "Délai dépassé — VPN lent ou serveur surchargé", "type": "RÉSEAU"}
    return {"code": "UNKNOWN", "msg": err[:200], "type": "AUTRE"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

### Lancer le backend

```bash
# → Dans index.html, remplacer les simulations par fetch('http://localhost:8000/api/execute', {...})
```

## ❓ QUESTIONS FRÉQUENTES

**Q : Où sont gardés les mots de passe si je ferme le navigateur ?**
> Les mots de passe sont dans le code HTML (en SHA-256 hashé). Ils persistent entre les sessions car ils sont dans le fichier. Les utilisateurs créés via l'interface existaient seulement en mémoire — à la fermeture du navigateur ils disparaissent. Solution permanente : backend avec base de données.

**Q : Comment ne pas oublier les mots de passe créés ?**
> Utiliser la section Admin → Référentiel des accès → Exporter. Stocker ce fichier Excel dans un endroit sécurisé (coffre SharePoint, OneDrive chiffré, coffre physique).

**Q : GitHub Pages c'est sécurisé pour une banque ?**
> Pour la demo UI et les tests : OUI. Pour la production bancaire avec vraies connexions BD : NON. Il faut un backend sur intranet AFG ou serveur privé. GitHub Pages expose le code source (sans les vraies clés si bien configuré).

**Q : Peut-on accéder à la plateforme depuis un téléphone ?**
> Oui. La v3.0 est responsive. Menu hamburger ☰ sur mobile. Toutes les fonctions disponibles. Optimal sur écran ≥ 5 pouces.

**Q : Est-ce que les tests sont sauvegardés si je ferme l'onglet ?**
> Non, actuellement tout est en mémoire navigateur. Pour la persistance, il faut le backend. En attendant, toujours exporter en Excel avant de fermer.
